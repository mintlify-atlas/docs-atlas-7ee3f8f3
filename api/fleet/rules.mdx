---
title: Fleet rules
description: Define and manage automated monitoring rules
---

The Fleet Rules resource allows you to create automated monitoring rules that can trigger actions based on conditions.

## List rules

Retrieve a paginated list of fleet rules.

```typescript
const rules = await client.fleet.rules.list(options?);
```

### Parameters

<ParamField path="options" type="ListRulesOptions" optional>
  Optional filtering and pagination parameters

  <ParamField path="enabled" type="boolean" optional>
    Filter rules by enabled status
  </ParamField>

  <ParamField path="limit" type="number" optional>
    Maximum number of rules to return per page
  </ParamField>

  <ParamField path="cursor" type="string" optional>
    Cursor for pagination
  </ParamField>
</ParamField>

### Returns

<ResponseField name="items" type="FleetRule[]">
  Array of fleet rules

  <ResponseField name="uid" type="string">
    Unique identifier for the rule
  </ResponseField>

  <ResponseField name="name" type="string">
    Rule name
  </ResponseField>

  <ResponseField name="description" type="string | null">
    Rule description
  </ResponseField>

  <ResponseField name="enabled" type="boolean">
    Whether the rule is enabled
  </ResponseField>

  <ResponseField name="condition" type="Record<string, unknown> | null">
    Rule condition configuration
  </ResponseField>

  <ResponseField name="actions" type="Array<Record<string, unknown>>">
    Actions to execute when rule is triggered
  </ResponseField>

  <ResponseField name="scope" type="Record<string, unknown> | null">
    Rule scope configuration
  </ResponseField>

  <ResponseField name="hitCount" type="number">
    Number of times the rule has been triggered
  </ResponseField>

  <ResponseField name="lastHitAt" type="string | null">
    Timestamp when the rule was last triggered
  </ResponseField>

  <ResponseField name="createdAt" type="string | null">
    Creation timestamp
  </ResponseField>

  <ResponseField name="updatedAt" type="string | null">
    Last update timestamp
  </ResponseField>
</ResponseField>

<ResponseField name="nextCursor" type="string | null">
  Cursor for the next page of results
</ResponseField>

<ResponseField name="previousCursor" type="string | null">
  Cursor for the previous page of results
</ResponseField>

<ResponseField name="hasMore" type="boolean">
  Whether more results are available
</ResponseField>

## Create rule

Create a new fleet monitoring rule.

```typescript
const rule = await client.fleet.rules.create(options);
```

### Parameters

<ParamField path="options" type="CreateRuleOptions" required>
  Rule creation details

  <ParamField path="name" type="string" required>
    Rule name
  </ParamField>

  <ParamField path="condition" type="Record<string, unknown>" required>
    Rule condition configuration defining when the rule triggers
  </ParamField>

  <ParamField path="actions" type="Array<Record<string, unknown>>" required>
    Actions to execute when the rule is triggered
  </ParamField>

  <ParamField path="description" type="string" optional>
    Rule description
  </ParamField>

  <ParamField path="enabled" type="boolean" optional>
    Whether the rule should be enabled (defaults to true)
  </ParamField>

  <ParamField path="scope" type="Record<string, unknown>" optional>
    Rule scope configuration to limit which devices/recordings this applies to
  </ParamField>
</ParamField>

### Returns

<ResponseField name="FleetRule" type="object">
  The newly created rule
</ResponseField>
